The memory that doesn't forget between sessions.

reverie is the neuroscience-grounded memory consolidation daemon for LLM coding harnesses. One Go binary serves MCP, HTTP on 127.0.0.1:7437, a CLI, and a TUI against a single SQLite + FTS5 store. It is the sole cross-harness memory of record for the estate — search observations, capture sessions, assemble context.

What it does

  • Observations — typed entries (decision, bug fix, discovery, convention) stored with topic_key paths for hierarchical browse
  • Full-text search — FTS5 over saved observations; partial, exact, and fuzzy match modes
  • Sessions — start, summarize, end; each harness session captured with prompts and key learnings
  • Context assembly — mem_context returns a formatted markdown blob (sessions + observations + prompts) primed for the next turn
  • Passive capture — mem_capture_passive parses ## Key Learnings headers straight into observations
  • Supersession — observations mark stale facts with a supersedes: field instead of deleting them
