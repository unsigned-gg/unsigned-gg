A full desktop the agent drives. The host never moves.

reach is an AI-drivable containerized desktop sandbox. An agent boots a complete desktop — browser, terminal, file system — inside an isolated container, operates it like a human would, and leaves nothing behind. GPU-ready, session-ephemeral, and container-native from the kernel up. The host tree is never touched.

What it does

Most agent sandboxes give the model a shell and call it a day. reach gives it a desktop. A real display server, a pointer, a keyboard, a browser able to click and read, a terminal able to type and wait, and a file system it can read and write — all inside a container the agent controls end to end. Tasks that need a GUI (filling a form, navigating a dashboard, verifying a rendered page) stop being hacks. They become just another action the agent takes.

Every session starts from a clean image and ends as nothing. There is no persistent state to leak, no leftover process to orphan, no host path to escape into. When the agent is done, the container is gone — only the artifacts it chose to emit survive.

Why a desktop, not a shell

  • See what the user sees — the agent gets frame captures and DOM reads, not just stdout. It can verify a button actually rendered before it clicks.
  • Drive real apps — browsers, IDEs, internal tools with no API. If a human can do the task on screen, the agent can do it in reach.
  • Isolate, don't emulate — it's a real container running real binaries, not a JS sandbox. CVEs and odd behaviors are the same ones production hits, not a mock's.
  • GPU when needed — model inference, rendering, video. Pass a GPU through to the container for one session; reclaim it when the session ends.
  • Throw the room away — session-ephemeral means each run is a clean image. No state drift, no "what did the last agent leave running," no forensic cleanup.

Status

Pre-alpha. The container + display + control layers are wired and a single agent can drive a browser through a real task. GPU passthrough works on a single node; multi-session scheduling is the active front. The host-tree invariant — nothing escapes the container unless the agent explicitly emits it — holds by construction and is the property reach will never trade away.
