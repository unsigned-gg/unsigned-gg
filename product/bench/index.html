Every score carries its receipts.

bench is the benchmark surface for LLM performance. A local bank of tasks lives next to a provenance-verified frontier leaderboard: every rank is backed by the exact prompt, model snapshot, runner, and environment that produced it. A SvelteKit PWA, installable and offline-capable, that treats reproducibility as the unit of trust.

What it does

  • Local bank — a versioned task bank you own. Prompts, expected outputs, scoring rubrics, and tool fixtures are content-addressed, so a run today can be checked against a run from any prior commit.
  • Frontier leaderboard — model rankings ordered by verified score. No anonymous submissions: every entry resolves to a provenance record signed against the bank revision it ran against.
  • Provenance metadata — model id and snapshot digest, prompt template + variables, runner version, host class, environment hash, and timestamps are captured per run and surfaced inline with the score.
  • Reproducible benchmarks — re-run a leaderboard entry from its provenance record. Drift between canonical and reproduced scores is shown as a delta, not hidden behind a single number.
  • PWA install — SvelteKit service worker, manifest, and offline cache keep the leaderboard and bank readable without a connection. Runs queue and sync when the network returns.
