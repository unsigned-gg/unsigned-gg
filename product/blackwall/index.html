The agent never touches your tree.

blackwall is the containment layer for AI agent execution. An agent runs inside an OS syscall jail; its output lands as a content-addressed custody record; nothing reaches your code until you settle it. One static Rust binary. 506 tests. Zero warnings.

v1.0.0 shipped

  • Custody core — records, changesets, worlds (DAG), typed settle, atomic-write, pure-Rust git ODB
  • Serve layer — axum REST API, JWT-SVID authz, MCP surface, multi-project isolation, JobExecutor (k8s Jobs)
  • Streaming — RunEvent bus, events.ndjson, box renderer, escalation sink, harness bridge
  • Telemetry — OTLP auto-fire (traces+logs+metrics), Langfuse ingestion, training corpus export
  • Orchestration — dispatch queue, DAG plans v2, batch merge, mid-run branch points, CRDT overlap merge
  • GitOps chain — reconcile → PR → merge-gate → watch daemon
  • Training data — run export-corpus --all produces joinable JSONL. AgentHER relabeling mines failures into gold.

Kubernetes orchestration

Static distroless binary (UID 1000, all caps dropped, seccomp RuntimeDefault). Helm chart, ArgoCD Application, and a JobExecutor that dispatches runs as one-shot k8s Jobs. The Landlock jail is verified active on Talos/Cygnus — it's the real security boundary, not RBAC.
