unsigned platform · agent containment

The agent never touches your tree.

blackwall is the containment layer for AI agent execution. An agent runs inside an OS syscall jail; its output lands as a content-addressed custody record; nothing reaches your code until you settle it. One static Rust binary.

The problem

An agent with write access is an unreviewed committer. It edits the working tree in place, mixes its changes with yours, and leaves you diffing your repository against your memory. Scale that to a fleet and the question stops being is the code good and becomes what actually happened.

blackwall answers the second question first. Execution is evidence before it is code.

Custody, not trust

Three invariants. Each one is enforced by a mechanism, not a convention.

jailConfinement

Landlock syscall confinement, native to the kernel. The agent works in a scratch materialization of your world — it never sees the real tree's path. Where the kernel can't enforce it, the run fails closed.

storeIdentity

Every run is captured whole — prompt, model, output, exit, usage — and addressed by the BLAKE3 hash of its content. The hash is the identity: the same run is the same object, and a tampered record is a different hash.

settleDecision

Output advances only through a typed state machine — select · release · apply · discard. A settled run is a moved value. Re-settling is a compile error, not a runtime check.

One run, end to end

Step through it. Every stage leaves an object in the store; the chain is replayable from any of them.

The provider seam is model-agnostic — any OpenAI-compatible gateway, the claude CLI, or the omp harness. Containment does not depend on which model is inside it.

Every run answers for itself

The trace ID is the run hash — the same hex identifies the custody object, the distributed trace, and the pull request it became. Every run auto-fires traces + logs + metrics over OTLP to your observability stack (Tempo, Loki, Prometheus, Arize Phoenix). The harness session file is captured inside the run's own directory, bound by a correlation id minted before the model says a word — the mapping between a run and everything it did is structural, not reported.

New in v1.0.0: every blackwall run is now a training data row.

otlpAuto-export

Traces + logs + metrics auto-fire on every run completion to the configured OTLP endpoint. No manual export needed — the data lake fills itself.

langfuseIngestion

Native Langfuse ingestion: trace-create + GENERATION observation-create pairs with prompt, completion, usage, cost, and custody metadata. One exporter, two backends.

corpusTraining data

blackwall run export-corpus --all emits every run as a JSONL row: prompt → completion → token usage → changeset paths → settle verdict → op trace. Directly usable for fine-tuning or eval. AgentHER relabeling mines failures into gold-standard training data.

Built for operators who run agents in fleets and answer for the result. If you need to know what an agent did, prove it, and undo it — that is custody, and it is the whole product.

Kubernetes orchestration

A static distroless binary, a Helm chart, an ArgoCD Application, and a JobExecutor that dispatches runs as one-shot k8s Jobs.

imageDistroless

Static musl binary in gcr.io/distroless/static-debian12:nonroot. UID 1000, all capabilities dropped, readOnlyRootFilesystem, seccomp RuntimeDefault. Never latest.

helmChart

Deployment, Service, SA (zero API perms), RBAC stub, NetworkPolicy, PDB. helm install blackwall deploy/helm/blackwall --set args='{serve}'

jobsExecutor

JobExecutor renders batch/v1 Job manifests (image, PVC mount, resource limits) and watches via kubectl. Runs execute as one-shot Jobs where Landlock is the security boundary, not RBAC.

The Landlock jail is verified active on Talos/Cygnus (CONFIG_SECURITY_LANDLOCK=y). Where the kernel can't enforce it, the run fails closed.
