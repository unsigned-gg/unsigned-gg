Every namespace, every binding, one pane.

k8s-audit is a SvelteKit PWA that turns a raw cluster-audit.json snapshot into a navigable dashboard. Namespaces, workloads, RBAC findings, Kyverno policies, IngressRoutes, ArgoCD apps, and the migration roadmap — all typed, filterable, and rendered client-side. A real-time view into the Cygnus production cluster and the Lyra dev cluster.

What it does

  • Cluster identity — current context, API server, provider, k8s version, CNI, and the Lyra→Cygnus migration target (bare-metal Talos, Cilium eBPF, WireGuard mTLS)
  • Namespaces — 46 namespaces classified by component (security, gitops, compute, networking, observability, data, ai, tenant, applications, system) with search + filter
  • Workloads — 69 deployments, 21 StatefulSets, 7 DaemonSets; image, replica count, type, search across all three
  • RBAC audit — 8 findings (1 CRITICAL, 2 HIGH, 3 MEDIUM, 2 LOW), ClusterRoleBindings, tenant RoleBindings, ResourceQuotas
  • Policy — 5 Kyverno ValidatingPolicies (all Audit mode), readiness + scope
  • Networking — 20 IngressRoutes with external hostnames, per-namespace grouping
  • GitOps — 58 ArgoCD applications, destination namespace, search
  • Roadmap — 4 migration phases (VKE → bare-metal Talos / Hetzner+Talos green) + 9 Linear tickets mapped to PRs
  • Nodes — 4-worker pool (VKE, migrating to bare-metal Talos hoplite-class nodes), version, OS, runtime

Architecture

Svelte 5 in runes mode, fully typed TypeScript data model, Bun as package manager + dev server, and adapter-static for a prerendered static build — deployable to any static host (Cloudflare Pages, Vercel, or a bare bun run scripts/serve.ts). The audit JSON is the single source of truth; the dashboard is a pure render of it.
