Every session, one pane of glass.

meridian is the read-first cockpit for a multi-session agent fleet. When six Claude and OMP sessions run at once, meridian is the surface that shows them all — session health, token spend, tool-call traces, and settle-state in a single live view. Internal, product-shaped, and built for the operator who answers for the fleet.

What it does

  • Fleet-wide session view — every active Claude/OMP session in one list, sorted by health and age. Stalled, erroring, and long-running sessions rise to the top; you never hunt for the one that quietly stopped.
  • Token spend, live — input/output/cached tokens per session and per fleet, with running cost against your model budget. Spot the session burning the day's quota before it burns through the night.
  • Tool-call traces — every tool invocation in order: name, arguments, result, duration. The audit trail that tells you what an agent actually did, not what it claims it did.
  • Settle-state visibility — pending, settled, and rolled-back changes per session, surfaced from blackwall custody. You see exactly which outputs have crossed the line into your tree and which are still held.
  • Read-first by design — meridian observes. It will not dispatch, settle, or mutate a run from the cockpit; the agent layer keeps the only write path. The operator sees everything; the operator acts elsewhere.

Architecture

meridian is a thin read surface over the estate's existing data plane. It does not own state — it joins signals that blackwall, reverie, and the Langfuse lake already produce. No new write path, no new source of truth, no duplicate custody record.

Why it exists

A fleet of autonomous agents is only as trustworthy as the operator's ability to see it. Run two sessions and you can watch them by hand. Run twelve, across providers and projects, and the gap between “what the agent said it did” and “what actually landed” becomes where the incident lives. meridian closes that gap: one read-first surface that shows every session's health, spend, traces, and settle-state, so the operator never answers for a fleet they could not see.
